LLM parameters

Sampling and generation

Controls randomness. Lower values (e.g. 0.1) produce focused, deterministic answers. Higher values (e.g. 1.0) make output more creative and varied.

Only considers tokens whose cumulative probability reaches this threshold. 1.0 = consider all tokens. Lower values (e.g. 0.9) cut unlikely tokens for more coherent output.

Limits the model to choosing among the top K most likely tokens at each step. Lower values reduce randomness; 40 is a good default.

Maximum number of tokens the model can generate for utility tasks (summarization, query rewriting). Conversation answers are unlimited and stop naturally. Default: 1024.

Reasoning

Chain-of-thought thinking

When enabled, the model thinks step-by-step before answering. Produces better results but uses more tokens and takes longer. The reasoning is shown in a collapsible block above the answer. Requires a model that supports thinking (e.g. Qwen3.5).

Maximum number of tokens the model can use for its internal chain-of-thought reasoning. Higher values allow deeper thinking but slow down responses. Only used when reasoning mode is enabled.

Search parameters

Web search and content extraction

How many times the model can issue follow-up searches to refine its answer. When this limit is reached, the user is prompted to continue or stop.

Maximum number of top-ranked URLs whose content will be fetched and indexed per search. Higher values give richer context but take longer.

Character limit for text extracted from each web page. Larger values capture more detail but may exceed the model's context window.

Number of concurrent workers fetching web pages. More workers speed up page retrieval but use more bandwidth and CPU.

Chat parameters

Follow-up conversation limits

Maximum characters of previous conversation context included when generating a follow-up reply. Larger values give the model more memory but use more of the context window.

Maximum number of previous messages to include as context for follow-up replies. Keeps conversations coherent without overloading the model.

Maximum number of top-ranked documents included as context when generating an answer. More documents give the model more material to cite but use more of the context window.

Number of search result cards shown initially. The rest are hidden behind a "show more" button.
